## 一、统一响应格式

### 1.1 规范要求

所有 API 接口必须返回统一 JSON 结构：

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0=成功，非0=失败 |
| msg | string | 提示信息，成功为"success"，失败为错误描述 |
| data | object/array/null | 业务数据，失败时为null |

### 1.2 错误码定义

| 错误码 | 含义 | HTTP状态码 |
|--------|------|-----------|
| 0 | 成功 | 200 |
| 1001 | 参数校验失败 | 400 |
| 1002 | 资源不存在 | 404 |
| 2001 | 飞书API调用失败 | 500 |
| 2002 | OCR识别失败 | 500 |
| 2003 | LLM分析失败 | 500 |
| 2004 | COS上传失败 | 500 |
| 9001 | 系统内部错误 | 500 |

### 1.3 代码实现

```java
// tech.jxing.returnvision.common.ResponseResult
package tech.jxing.returnvision.common;

import lombok.Data;

@Data
public class ResponseResult<T> {
    private int code;
    private String msg;
    private T data;

    public static <T> ResponseResult<T> success(T data) {
        ResponseResult<T> result = new ResponseResult<>();
        result.setCode(0);
        result.setMsg("success");
        result.setData(data);
        return result;
    }

    public static <T> ResponseResult<T> error(int code, String msg) {
        ResponseResult<T> result = new ResponseResult<>();
        result.setCode(code);
        result.setMsg(msg);
        result.setData(null);
        return result;
    }
}
```

### 2.2 全局异常处理器

```java
// tech.jxing.returnvision.common.GlobalExceptionHandler
package tech.jxing.returnvision.common;

import tech.jxing.returnvision.common.exception.BizException;
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    /** 业务异常：返回具体错误码 */
    @ExceptionHandler(BizException.class)
    public ResponseResult<?> handleBizException(BizException e) {
        log.warn("[业务异常] code={}, msg={}", e.getCode(), e.getMessage());
        return ResponseResult.error(e.getCode(), e.getMessage());
    }

    /** 参数校验异常：返回1001 */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseResult<?> handleValidation(MethodArgumentNotValidException e) {
        String msg = e.getBindingResult().getFieldErrors().stream()
                .map(f -> f.getField() + ": " + f.getDefaultMessage())
                .reduce((a, b) -> a + "; " + b)
                .orElse("参数校验失败");
        return ResponseResult.error(1001, msg);
    }

    /** 系统异常：返回9001 */
    @ExceptionHandler(Exception.class)
    public ResponseResult<?> handleException(Exception e) {
        log.error("[系统异常]", e);
        return ResponseResult.error(9001, "系统内部错误");
    }
}
```

## 三、日志配置

### 3.1 规范要求

- 统一日志格式：`[时间] [级别] [模块名] 消息`
- 业务日志使用中文
- 日志文件按天轮转，保留30天
- 使用 SLF4J + Logback（Spring Boot 默认）

### 3.2 Logback 配置

```xml
<!-- src/main/resources/logback-spring.xml -->
<configuration>
    <property name="LOG_PATH" value="logs"/>
    <property name="LOG_PATTERN" value="[%d{yyyy-MM-dd HH:mm:ss}] [%level] [%logger{36}] %msg%n"/>

    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder><pattern>${LOG_PATTERN}</pattern></encoder>
    </appender>

    <!-- 文件输出（按天轮转，保留30天） -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/return-ocr.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_PATH}/return-ocr.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder><pattern>${LOG_PATTERN}</pattern></encoder>
    </appender>

    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
    </root>
</configuration>
```

## 十、注释规范

### 10.1 核心原则

注释的目标：**让代码审查者通过注释就能理解业务流向和实现步骤，无需逐行读代码。**

### 10.2 类级注释模板

```java
/**
 * 【退货上传】上传识别控制器
 *
 * 职责：接收前端图片上传，调度OCR识别全流程，返回结构化结果
 * 层级：Controller 层
 * 路由：/api/upload
 */
```

### 10.3 Controller 方法注释（业务流向图）

Controller 方法的注释是**最重要的注释**，必须完整描述业务流向，让审查者一眼看懂整个请求的生命周期：

```java
/**
 * 上传快递单图片并识别
 *
 * 业务流程：
 *   1. 接收前端上传的快递单图片（multipart/form-data）
 *   2. 上传图片至腾讯云COS，获取公开访问URL
 *   3. 并行调用双引擎OCR：
 *      ├─ 引擎A：智谱OCR（layout_parsing API）
 *      └─ 引擎B：阿里云面单OCR（RecognizeWaybill API）
 *   4. 交叉验证引擎逐字段比对两引擎结果
 *      ├─ 一致 → 自动采用（高置信）
 *      ├─ 部分差异 → 标记差异字段
 *      └─ 完全不一致 → 转人工
 *   5. DeepSeek LLM分析：语义校验 + 退货原因提取 + 智能分类
 *   6. 识别结果存入MySQL（status=pending）
 *   7. 返回识别结果 + 交叉验证状态 + LLM分析
 *
 * @param file 快递单图片
 * @return ResponseResult 含识别结果、交叉验证状态、LLM分析
 */
@PostMapping("/api/upload")
public ResponseResult<UploadResponse> upload(@RequestParam("file") MultipartFile file) {
    // 步骤1：上传COS
    String imageUrl = cosClientService.uploadToCos(file.getBytes(), filename);
    // 步骤2：双引擎OCR + 交叉验证
    Map<String, Object> crossResult = ocrCrossValidatorService.dualEngineOcr(imageUrl);
    // 步骤3：LLM分析
    Map<String, Object> llmResult = llmAnalyzerService.analyzeWithLlm(crossResult);
    // 步骤4：存入MySQL
    recordManagerService.savePendingRecord(crossResult, llmResult, imageUrl);
    // 步骤5：返回结果
    return ResponseResult.success(buildResponse(crossResult, llmResult));
}
```

```java
/**
 * 双引擎并行OCR + 交叉验证
```

### 10.5 注释规则总结

| 位置 | 注释内容 | 必填 |
|------|---------|------|
| **类** | 模块名 + 职责 + 层级 + 路由（Controller） | ✅ |
| **Controller方法** | 业务流程（编号步骤）+ 参数 + 返回值 | ✅ |
| **Service方法** | 实现步骤（编号步骤）+ 参数 + 返回值 | ✅ |
| **方法内** | 每个步骤前加 `// 步骤N：描述` | ✅ |
| **复杂逻辑** | 行内注释解释为什么这样做 | 按需 |
| **简单getter/setter** | 不需要注释 | ❌ |

### 10.6 注释语言

- 所有注释使用**中文**
- 代码中的变量名、方法名使用**英文**
- 日志消息使用**中文**
