## 七、代码实现规范与模块拆分

> 沿用统一规范：单一职责、独立可测、业务注释、日志即流程、配置驱动。

### 7.1 模块流程编号

| 流程 | 模块 | 职责 | 可独立验证 |
|------|------|------|-----------|
| ① | controller/ | 接收上传请求，返回识别结果 | curl/浏览器上传图片测试 |
| ② | service/CosClientService.java | 腾讯云COS图片上传 | 上传图片，验证返回URL可访问 |
| ② | service/OcrZhipuService.java | 智谱OCR识别（引擎A） | 传入图片URL，验证识别结果 |
| ② | service/OcrAliyunService.java | 阿里云面单OCR识别（引擎B） | 传入测试图片，验证识别结果 |
| ② | service/OcrCrossValidatorService.java | 双引擎交叉验证 | 传入两引擎结果，验证比对逻辑 |
| ② | service/LlmAnalyzerService.java | DeepSeek LLM分析 | 传入OCR结果，验证语义校验+退货原因+分类 |
| ② | service/ValidatorService.java | 数据校验 | 传入测试数据，验证校验逻辑 |
| ③ | feishu/FeishuClientService.java | 写入飞书表格 | 调用API，验证记录已写入 |

### 7.2 分阶段交付建议

| 阶段 | 交付模块 | 验证方式 |
|------|---------|---------|
| 第1阶段 | service/CosClientService.java | 上传图片到COS，验证URL可访问 |
| 第2阶段 | service/OcrZhipuService.java + OcrAliyunService.java | 传入COS URL，验证双引擎OCR识别结果 |
| 第3阶段 | service/OcrCrossValidatorService.java + LlmAnalyzerService.java | 传入两引擎结果，验证交叉验证+LLM分析逻辑 |
| 第4阶段 | model/ + feishu/FeishuClientService.java + feishu/FeishuBotService.java | 验证MySQL建表 + 飞书写入（含photo_url）+ 机器人通知 |
| 第5阶段 | service/ValidatorService.java + RecordManagerService.java | 验证校验与记录管理 |
| 第6阶段 | controller/ + ReturnVisionApplication.java | 启动应用，端到端上传->COS->双引擎->交叉验证->LLM分析->确认->写入 |
| 第7阶段 | frontend/ | 前端页面，拍照上传+差异高亮+AI分析展示+确认 |
