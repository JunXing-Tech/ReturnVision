# 实施计划与风险

## 八、实施计划

### 阶段一：服务准备
1. 智谱开放平台注册，获取 API Key，开通OCR解析工具服务
2. 阿里云开通电子面单OCR服务，获取 AccessKey（200次/月免费额度）
3. 腾讯云开通COS对象存储，创建Bucket，获取 SecretId/SecretKey
4. DeepSeek开放平台注册，获取 API Key（deepseek-v4-flash）
5. MySQL 8.0 数据库准备，创建数据库
6. 飞书创建多维表格，按 04-详细方案设计.md 4.3 节设计表结构（含return_reason、return_category字段）
7. 飞书群创建自定义机器人，获取 Webhook URL
8. 验证全部API连通性（各调用一次测试）

### 阶段二：后端开发
1. 实现腾讯云COS图片上传模块
2. 实现智谱OCR识别模块（layout_parsing API + JSON Schema）
3. 实现阿里云面单OCR识别模块（含valueProb置信度提取）
4. 实现双引擎交叉验证引擎（并行调用 + 字段比对 + 差异标记）
5. 实现DeepSeek LLM分析模块（语义校验 + 退货原因 + 智能分类）
6. 实现数据校验逻辑（格式校验 + 交叉验证状态检查）
7. 实现MySQL数据库模型（return_records + ocr_log）
8. 实现飞书表格写入 + 机器人通知
9. 实现API接口

### 阶段三：前端开发
1. 图片上传组件
2. 识别结果展示与编辑（含差异字段高亮）
3. 交叉验证状态展示（一致/差异/人工录入）
4. 确认写入交互
5. 记录列表页

### 阶段四：联调与部署
1. 端到端联调（拍照->双引擎OCR->交叉验证->确认->飞书）
2. 双引擎交叉验证准确率测试（抽样100张面单验证）
3. 部署上线

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 智谱OCR识别不准 | 运单号错误 | 双引擎交叉验证；引擎B互补；人工确认环节 |
| 阿里云OCR识别不准 | 运单号错误 | 双引擎交叉验证；valueProb置信度过滤；人工确认 |
| 双引擎结果不一致 | 需人工介入 | 高亮差异字段，客服只核对差异部分；~10%场景 |
| 快递单照片模糊 | 识别失败 | 前端拍照引导提示；失败转手动录入 |
| 飞书API限频 | 写入失败 | 批量写入，50次/秒足够 |
| 智谱OCR费用 | 成本增加 | 0.01元/次，月1000单仅10元，成本极低 |
| 阿里云OCR费用 | 成本增加 | 200次/月免费额度；超出0.0825元/次 |
| 腾讯云COS费用 | 成本增加 | 50GB/月免费额度（6个月）；退货照片约0.5MB/张 |
| MySQL数据库 | 运维依赖 | 公司IT统一运维；定期备份 |
| 智谱OCR并发限制 | 批量处理慢 | 最大并发2，批量上传时排队处理 |
| 面单版式多样 | 部分无法识别 | 智谱支持复杂版面+手写体；阿里云专用面单优化 |
