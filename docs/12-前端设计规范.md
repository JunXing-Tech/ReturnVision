# 12 - 前端设计规范

---

## 〇、文档定位

- 当前项目没有专门的前端规范，[基础框架规范.md](./基础框架规范.md) 与 [07-项目结构与代码规范.md](./07-项目结构与代码规范.md) 均只覆盖后端。
- 本文件**把现有前端已经落地的约定固化下来**，作为后续新增页面、优化交互的唯一依据。
- 与 AGENTS.md「编码约定」不冲突处沿用；与本文件冲突处，前端以**本文件**为准。

## 一、技术栈（固定项，不得擅自更换）

| 项 | 选型 | 备注 |
|----|------|------|
| 框架 | Vue 3（`<script setup>` 组合式 API） | 已落地 |
| 构建 | Vite 5 | 已落地 |
| 组件库 | Element Plus 2.7 | 通过覆盖 `--el-*` 变量做主题，**不 fork、不 patch 组件库** |
| HTTP | axios | 仅用于普通请求；**SSE 用浏览器原生 `fetch`** |
| 路由 | 无 vue-router | 用 `v-show` 切换 Panel，保留组件状态 |
| 状态管理 | 无 Pinia/Vuex | 跨页状态用 `App.vue` 的 `ref` + props/emit 传递 |
| 图标 | `@element-plus/icons-vue`，经 `src/icons.js` 中转 | 避免散落 import |

**禁止引入**：新组件库、vue-router、Pinia/Vuex、Tailwind 等额外样式系统 —— 确需则走 docs/11 第五章"门禁"先讨论。

## 二、目录与文件组织

```
returnvision-frontend/src/
├── App.vue               # 全局：布局壳 + 设计 token(:root) + 路由切换
├── main.js               # 入口
├── api.js                # 【唯一 API 收口】所有后端调用集中在此
├── icons.js              # 图标中转，组件统一从此 import
├── components/
│   ├── DashboardPanel.vue
│   ├── RecognitionPanel.vue
│   └── RecordsPanel.vue
└── composables/
    └── useMobile.js      # 响应式断点等可复用逻辑
```

约定：
- 一个 Tab / 一类页面 = 一个 `*Panel.vue`，粒度不再细拆子组件目录，除非确实复杂
- 公共复用逻辑放 `composables/`，纯函数工具放 `utils/`（暂无）
- 入参/出参跨页面通过 `App.vue` `ref` + `emit`，**不引入全局 store**

## 三、设计系统 Token（来源：`App.vue :root`）

> 改这一节等于改全站，必须走 docs/11 第五章门禁"先讨论"。

### 3.1 颜色（单色系，对齐 Vercel 风格）

```css
--color-bg: #ffffff;
--color-card: #ffffff;
--color-muted: #f5f5f5;
--color-fg: #121212;
--color-fg-muted: #6b6b6b;
--color-primary: #121212;
--color-primary-fg: #ffffff;
--color-secondary: #f5f5f5;
--color-accent: #f5f5f5;
--color-border: #e8e8e8;
--color-input: #e8e8e8;
--color-success / --color-success-strong / --color-success-subtle;
--color-error / --color-error-strong / --color-error-subtle;
--color-sidebar / --color-sidebar-fg / --color-sidebar-border / --color-sidebar-accent;
--color-chart-2 / --color-chart-3;
```

### 3.2 排版与尺度
- 圆角：`--radius: 8px`
- 间距单位：`--spacing: 4px`，间距一律用 `calc(var(--spacing) * N)`
- 阴影：`--shadow-sm`、`--shadow-md`
- 过渡：`--transition: 150ms ease`
- 字体：`--font-sans: 'Inter', system-ui, -apple-system, 'Microsoft YaHei', sans-serif`
- 等宽：`--font-mono: 'JetBrains Mono', 'SF Mono', 'Consolas', monospace`
- 基准字号：`13px`（密集工作台风格）

### 3.3 Element Plus 主题覆盖
通过覆盖 `--el-*` 变量统一主题（见 `App.vue`），例如：
```css
--el-color-primary: #121212;
--el-border-color: #e8e8e8;
--el-border-radius-base: 8px;
--el-font-size-base: 13px;
```
> 禁止用 `::v-deep` 大面积改 Element Plus 组件样式；能用变量覆盖的优先用变量。

## 四、页面与组件粒度

### 4.1 外壳统一 `screen-shell`
每个 `*Panel.vue` 顶层模板：
```html
<div class="screen-shell">
  <section class="hero"> ... </section>
  <section class="card"> ... </section>
</div>
```

### 4.2 内部分区习惯（参考 `DashboardPanel.vue`）
- `hero`：标题 + 主操作按钮
- `kpi-grid`：关键指标卡片网格
- `main-content`：主内容区（左表 + 右图）
- `card`：分组容器，内含 `card-header` + 内容
- `toolbar`：筛选/搜索/刷新

### 4.3 路由切换
- `App.vue` 维护 `activeTab`，三种 Tab 用 `v-show` 切换（**不用 v-if**，保留上传进度等状态）
- 跨页传值：`pendingEditRecord` 等 `ref`，通过 props/emit 携带
- 顶栏"搜索框"目前为占位，接入搜索功能时统一抽到 `composables/`

## 五、模板注释规范（与后端风格一致）

- 模板内用 **`<!-- 步骤N：描述 -->`** 标注大区块，与后端 `// 步骤N` 对齐
- 注释一律**中文**，`<script setup>` 顶部按"步骤N"说明逻辑分段
- 组件文件头加一句话职责注释

示例：
```html
<!-- 步骤1：模式切换 + 多态视图 -->
<div class="screen-shell">
  <!-- 步骤2：空闲态上传区 -->
  ...
  <!-- 步骤3：处理态 SSE 时间线 -->
  ...
</div>
```

```js
// 步骤4：状态机：idle / processing / done / error
const state = ref('idle');
```

## 六、样式书写约定

| 项 | 约定 |
|----|------|
| 颜色 | 一律 `var(--color-*)`，**禁止硬编码 `#xxxxxx`** |
| 间距 | 一律 `calc(var(--spacing) * N)`，**禁止魔法数 px** |
| 圆角 | 一律 `var(--radius)`，特殊形状除外 |
| 布局 | 优先 Flex / Grid，少用 float/position |
| 作用域 | 组件 `<style>` 默认 scoped；全局样式只放 `App.vue` |
| 响应式 | 通过 `useMobile` + `@media` 断点（`1080px`、`900px`） |
| 动效 | 用 `var(--transition)`，禁止花哨动画；加载用 spinner |

> **不得新增全局样式文件**（如 `main.css`、`theme.scss`），所有 token 集中在 `App.vue :root`。

## 七、API 调用约定

### 7.1 收口
- **所有请求集中在 `src/api.js`**，组件只调用其导出方法，**禁止在组件里裸 import axios / fetch**
- `api.js` 内 `axios` 实例 `baseURL='/api'`，超时按场景设（普通 120s、批量 300s）

### 7.2 响应解包
后端统一 `ResponseResult<T>`，但前端历史代码同时读驼峰与下划线（见 RecordsPanel），新代码请：
- 优先读取后端返回的 `data` 字段并直接消费
- 字段命名以后端实体为准，前端展示层做兼容映射，不脏化全局

### 7.3 SSE（流式上传）
- 必须用**原生 `fetch`** 读 `ReadableStream`，按 `data:` 行解析（见 `api.js` 的 `uploadSSE`）
- 提供 `onStep / onResult / onError` 回调与 `AbortController` 取消
- **不要**用 axios 发 SSE；鉴权一旦接入，token 走 Header 或 URL 拼参（fetch 不能用 axios 拦截器）

### 7.4 错误处理
- 统一在 `api.js` 不捕获，由调用方在 `try/catch` 或 `.catch` 处理，给出 toast/inline 错误
- 业务失败展示后端 `msg`；网络错误给友好文案

---

## 八、表单 / 校验 / 提交态

- 用 `el-form` + `:model` + `:rules`，校验规则集中在组件 `const rules = {...}`
- 提交按钮必须有 loading 态（`:disabled="submitting"` + spinner）
- 关键操作（删除、批量确认）必须二次确认（`ElMessageBox.confirm`）
- 成功/失败用 `ElMessage` 反馈，与 `--color-success / --color-error` 视觉一致

---

## 九、表格约定（密集工作台风格）

- 容器 `table-wrap` + `data-table`，密集行高，13px 字号
- 状态列用 `pill`（`pill-pending` / `pill-synced` 等）
- 工具栏统一结构：状态筛选（el-select）+ 搜索框（`search-shell`）+ 刷新（`icon-btn`）
- 大量数据后续若超 500 行再考虑虚拟滚动，先不过度设计

---

## 十、响应式与移动端

- 断点：`≤1080px` 收窄搜索框；`≤900px` 隐藏侧栏，单列布局
- 复用 `composables/useMobile.js`
- 移动端上传需触发系统相机：`<input accept="image/*" capture="environment">`，按需开启

---

## 十一、可访问性（最低要求）

- 按钮用 `<button>` 而非 `<div @click>`，保留键盘可达
- 图标按钮加 `aria-label` 或旁置文案
- 颜色不作为唯一信息载体（状态除颜色外用文字 pill）
- 表单 label 与控件绑定

---

## 十二、新增页面的标准脚手架

> 新增页面时，复制此骨架起步，再按业务填充，避免结构游离。

```vue
<template>
  <!-- 步骤1：页面外壳 -->
  <div class="screen-shell">
    <!-- 步骤2：Hero -->
    <section class="hero">
      <div class="eyebrow">分类</div>
      <h1 class="hero-title">页面标题</h1>
      <p class="hero-subtitle">一句话说明</p>
    </section>

    <!-- 步骤3：主内容卡片 -->
    <section class="card">
      <div class="card-header">
        <div class="card-heading">
          <div class="eyebrow">数据</div>
          <h3 class="card-title">主标题</h3>
        </div>
        <div class="toolbar">
          <!-- 筛选 / 搜索 / 刷新 -->
        </div>
      </div>
      <div v-loading="loading" class="table-wrap">
        <!-- 表格或内容 -->
      </div>
    </section>
  </div>
</template>

<script setup>
// 步骤4：状态与 API 调用
import { ref, onMounted } from 'vue';
import api from '../api';

const loading = ref(false);
const list = ref([]);

// 步骤5：加载方法
const load = async () => {
  loading.value = true;
  try {
    const { data } = await api.getRecords();
    list.value = data?.list || [];
  } catch (e) {
    // 错误文案展示
  } finally {
    loading.value = false;
  }
};

onMounted(load);
</script>

<style scoped>
/* 步骤6：组件作用域样式，引用全局 token */
.foo { padding: calc(var(--spacing) * 4); border-radius: var(--radius); color: var(--color-fg); }
</style>
```

注册到 `App.vue`：
```js
import NewPanel from './components/NewPanel.vue';
const tabs = [ ..., { key: 'new', label: '新页面', icon: SomeIcon }];
// 模板：<NewPanel v-show="activeTab === 'new'" />
```

## 十三、禁止事项（前端专属）

| 禁止 | 原因 |
|------|------|
| ❌ 在组件里裸用 axios / fetch | 必须经 `api.js` 收口 |
| ❌ 硬编码颜色 / 魔法数 px | 必须用 token |
| ❌ 引入新组件库 / 路由库 / 状态库 | 默认拒绝，先讨论 |
| ❌ 用 `v-if` 切主路由 Panel | 应 `v-show` 保留状态 |
| ❌ 跳过模板"步骤N"注释 | 与后端注释规范一致 |
| ❌ 用 SSE 加 axios 拦截器 | fetch 流式 displeases axios |
| ❌ 全局样式散落到多个文件 | token 集中 `App.vue :root` |
