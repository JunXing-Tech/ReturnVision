## 三、设计系统 Token（来源：`App.vue :root`）

> 改这一节等于改全站，必须走 docs/11 第五章门禁"先讨论"。
> v2.1 起采用 **Vercel 单色系双主题**：默认浅色（运营场景优先），暗色通过 `:root.dark` 覆盖。
> 组件一律用 `var(--color-*)`，**禁止写死色值**--切换主题时组件零改动。

### 3.1 双主题架构

- `:root` 定义浅色为默认值（全量 token）
- `:root.dark` 仅覆盖需要变化的色值（基础面/主色/边框/状态色/侧边栏/图表色/阴影/Element Plus 变量）
- 尺度变量（`--radius`/`--spacing`/`--font-*`/`--transition`）两主题共享，不重复定义
- 通过给 `<html>` 加/去 `dark` class 切换（见第十五章）

### 3.2 颜色对照（浅色默认 / 暗色覆盖）

| Token | 浅色（默认） | 暗色（:root.dark） | 用途 |
|-------|-------------|-------------------|------|
| `--color-bg` | `#ffffff` | `#0a0a0a` | 全站背景 |
| `--color-card` | `#ffffff` | `#111111` | 卡片/容器底 |
| `--color-muted` | `#f5f5f5` | `#1a1a1a` | 次级容器/强调态底 |
| `--color-fg` | `#0a0a0a` | `#fafafa` | 主文字 |
| `--color-fg-muted` | `#6b6b6b` | `#a1a1a1` | 次级文字/标签 |
| `--color-primary` | `#0a0a0a` | `#fafafa` | 主色（按钮底） |
| `--color-primary-fg` | `#ffffff` | `#171717` | 主色上的文字 |
| `--color-primary-hover` | `#2a2a2a` | `#e0e0e0` | 主色按钮 hover |
| `--color-accent` | `#f5f5f5` | `#1a1a1a` | hover/激活态底 |
| `--color-border` | `#e8e8e8` | `#262626` | 1px 细边框 |
| `--color-input` | `#e8e8e8` | `#262626` | 输入框边框 |
| `--color-success` | `#16a34a` | `#62d178` | 状态色点/文字 |
| `--color-error` | `#dc2626` | `#ff6166` | 状态色点/文字 |
| `--color-warning` | `#d97706` | `#f5a623` | 状态色点/文字 |
| `--color-success-subtle` | `#f0fdf4` | `#1a2e1f` | 小徽章深底 |
| `--color-error-subtle` | `#fef2f2` | `#2e1a1c` | 小徽章深底 |
| `--color-sidebar` | `#fafafa` | `#0a0a0a` | 侧边栏底 |
| `--color-chart-1~5` | 浅蓝阶 | 亮蓝阶 | 图表单色蓝 |

**状态色使用铁律**：状态指示用 `w-2 h-2 rounded-full` 小色点 + 灰阶文字描述，**禁止**用状态色做整块背景填充（如整行高亮、整块红色卡片）。仅 `success-subtle/error-subtle` 这类深底用于小徽章底色。两主题均适用。

### 3.3 排版与尺度（两主题共享）
- 圆角：`--radius: 8px`（卡片用 12px 时显式写 `border-radius: 12px`）
- 间距单位：`--spacing: 4px`，间距一律用 `calc(var(--spacing) * N)`
- 阴影：浅色用 `--shadow-sm/md`（半透明黑 0.04~0.06），暗色覆盖为 0.3~0.4
- 过渡：`--transition: 150ms ease`
- 字体：`--font-sans: 'Inter', 'Geist', system-ui, -apple-system, 'Microsoft YaHei', sans-serif`
- 等宽：`--font-mono: 'Geist Mono', 'JetBrains Mono', 'SF Mono', 'Consolas', monospace`
- 基准字号：`13px`（密集工作台风格）；运单号/数字/时间戳一律用 `var(--font-mono)` 强化数据感
- 标题字距：大标题加 `letter-spacing: -0.01em`（tracking-tight，Vercel 标志）

### 3.4 Element Plus 主题覆盖
两主题分别在 `:root` 和 `:root.dark` 覆盖 `--el-*` 变量，关键项：
```css
/* 浅色 */
--el-color-primary: #0a0a0a;
--el-bg-color: #ffffff;
--el-text-color-primary: #0a0a0a;
--el-border-color: #e8e8e8;
--el-fill-color: #f5f5f5;

/* 暗色（:root.dark 覆盖） */
--el-color-primary: #fafafa;
--el-bg-color: #111111;
--el-text-color-primary: #fafafa;
--el-border-color: #262626;
--el-fill-color: #1a1a1a;
```
> 禁止用 `::v-deep` 大面积改 Element Plus 组件样式；能用变量覆盖的优先用变量。
> 两个主题块都要覆盖 `--el-fill-color-*`、`--el-mask-color`、`--el-box-shadow` 等浮层变量，详见 App.vue。

## 六、样式书写约定

| 项 | 约定 |
|----|------|
| 颜色 | 一律 `var(--color-*)`，**禁止硬编码 `#xxxxxx`**；暗色下状态色只做小色点/文字色，不整块填色 |
| 间距 | 一律 `calc(var(--spacing) * N)`，**禁止魔法数 px** |
| 圆角 | 一律 `var(--radius)`（8px），卡片级容器可显式 12px，特殊形状除外 |
| 布局 | 优先 Flex / Grid，少用 float/position |
| 作用域 | 组件 `<style>` 默认 scoped；全局样式只放 `App.vue` |
| 响应式 | 通过 `useMobile` + `@media` 断点（`1080px`、`900px`） |
| 动效 | 用 `var(--transition)`，禁止花哨动画；加载用 spinner |
| 数据字体 | 运单号/数字/时间戳用 `var(--font-mono)`，强化「数据 vs 叙述」层级 |
| 标题字距 | 大标题（hero-title/card-title 等）加 `letter-spacing: -0.01em` |

> **不得新增全局样式文件**（如 `main.css`、`theme.scss`），所有 token 集中在 `App.vue :root`。

## 七、API 调用约定

### 7.1 收口
- **所有请求集中在 `src/api.js`**，组件只调用其导出方法，**禁止在组件里裸 import axios / fetch**
- `api.js` 内 `axios` 实例 `baseURL='/api'`，超时按场景设（普通 120s、批量 300s）

### 7.2 响应解包
后端统一 `ResponseResult<T>`，但前端历史代码同时读驼峰与下划线（见 RecordsPanel），新代码请：
- 优先读取后端返回的 `data` 字段并直接消费
- 字段命名以后端实体为准，前端展示层做兼容映射，不脏化全局

### 7.3 SSE（流式上传）
- 必须用**原生 `fetch`** 读 `ReadableStream`，按 `data:` 行解析（见 `api.js` 的 `uploadSSE`）
- 提供 `onStep / onResult / onError` 回调与 `AbortController` 取消
- **不要**用 axios 发 SSE；鉴权一旦接入，token 走 Header 或 URL 拼参（fetch 不能用 axios 拦截器）

### 7.4 错误处理
- 统一在 `api.js` 不捕获，由调用方在 `try/catch` 或 `.catch` 处理，给出 toast/inline 错误
- 业务失败展示后端 `msg`；网络错误给友好文案

## 八、表单 / 校验 / 提交态

- 用 `el-form` + `:model` + `:rules`，校验规则集中在组件 `const rules = {...}`
- 提交按钮必须有 loading 态（`:disabled="submitting"` + spinner）
- 关键操作（删除、批量确认）必须二次确认（`ElMessageBox.confirm`）
- 成功/失败用 `ElMessage` 反馈，与 `--color-success / --color-error` 视觉一致

## 九、表格约定（密集工作台风格）

- 容器 `table-wrap` + `data-table`，密集行高，13px 字号
- 状态列用「小色点 + 文字」徽章（Vercel 风格）：`inline-flex items-center gap-1.5`，底色用 `var(--color-accent)`，靠 `w-1.5 h-1.5 rounded-full` 色点区分状态（success/warning/error），文字保持灰阶
  - 已同步：`var(--color-success)` 色点
  - 待确认：`var(--color-warning)` 色点
  - 识别失败：`var(--color-error)` 色点
- 运单号列一律 `font-family: var(--font-mono)`，强化数据感
- 表头用 `text-transform: uppercase; letter-spacing: 0.05em` 小标签风格，颜色 `var(--color-fg-muted)`
- 行 hover 用 `var(--color-muted)` 透明叠加，禁止整行状态色高亮
- 工具栏统一结构：状态筛选（el-select）+ 搜索框（`search-shell`）+ 刷新（`icon-btn`）
- 大量数据后续若超 500 行再考虑虚拟滚动，先不过度设计

## 十三、禁止事项（前端专属）

| 禁止 | 原因 |
|------|------|
| ❌ 在组件里裸用 axios / fetch | 必须经 `api.js` 收口 |

## 十五、主题切换规范（v2.1 新增）

### 15.1 架构
- 主题切换通过给 `<html>` 加/去 `dark` class 实现，触发 `:root.dark` CSS 覆盖
- 禁止用 JS 直接修改 `--color-*` 变量值；禁止在组件内写 `@media (prefers-color-scheme)` 分支
- 组件一律用 `var(--color-*)`，切换时组件零改动

### 15.2 composable
- 主题逻辑集中在 `src/composables/useTheme.js`，组件只调用 `useTheme()` 取 `isDark` 和 `toggleTheme`
- 重复调用 `useTheme()` 安全（内部 `initialized` 守卫，只初始化一次）

### 15.3 首次访问策略
- **localStorage 优先**：键名 `returnvision-theme`，值为 `'light'` 或 `'dark'`
- 未保存时**跟随系统** `prefers-color-scheme`
- 用户手动切换后写入 localStorage，之后不再跟随系统变化（系统主题变化监听器检查 localStorage，有值则忽略）

### 15.4 防闪屏（FOUC）
- `main.js` 在 `createApp` 前提前读取主题并应用 `dark` class，避免挂载前的浅色闪屏
- 该逻辑与 `useTheme.js` 的 `resolveInitialTheme` 保持一致，改动时两处同步

### 15.5 切换按钮
- 位置：顶栏右侧（搜索框旁的工具栏），与通知铃铛同排
- 图标：浅色态显示月亮（Moon，提示可切到暗色），暗色态显示太阳（Sun）
- 样式复用 `.icon-btn`，hover 用 `var(--color-accent)`，两主题自动跟随
- 必须有 `aria-label`（"切换到浅色"/"切换到暗色"），保证可访问性

### 15.6 新增主题色值的原则
- 两主题下都要保证文字与背景对比度（WCAG AA：正文 ≥ 4.5:1）
- 暗色下状态色需提亮（如 success `#16a34a` → `#62d178`），避免暗底上发闷
- 新增 token 必须同时在 `:root` 和 `:root.dark` 定义，缺一会在某主题下回退到默认值
