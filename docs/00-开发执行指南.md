## 分步执行计划

### 步骤 1：项目骨架搭建 ✅

**目标**：搭建后端 Spring Boot 3.x + 前端 Vue 3 项目骨架，能启动空服务。

**参考**：07-项目结构与代码规范.md

**任务**：
1. 创建 `returnvision-backend/` Maven 项目结构（controller/service/feishu/model/config）
2. 创建 `returnvision-frontend/` Vue 3 + Vite 项目
3. 安装依赖（见 08 环境依赖）
4. `ReturnVisionApplication.java` 能启动 Spring Boot，访问 `/` 返回 `{"status": "ok"}`
5. `application.yml` 配置模板（API Key 占位符）

**交付**：项目可启动，目录结构完整。

> ⏸️ **检查点 1**：骨架搭建完成后暂停，用户检查目录结构是否符合 07 规范。

---

### 步骤 2：数据库模型

**目标**：实现 MySQL 数据库建表 + MyBatis-Plus Entity。

**参考**：05-数据库设计.md（完整参照）

**任务**：
1. `config/AppConfig.java`：MyBatis-Plus 配置 + MySQL 数据源连接
2. `model/entity/ReturnRecord.java` + `model/entity/OcrLog.java`：MyBatis-Plus Entity 类
3. `model/mapper/ReturnRecordMapper.java` + `model/mapper/OcrLogMapper.java`：Mapper 接口
4. 验证：启动后自动建表，表结构与 05 文档一致（含 image_url 字段）

**交付**：MySQL 数据库可初始化，两张表创建成功。

> ⏸️ **检查点 2**：数据库建表完成后暂停，用户检查表结构。

---

### 步骤 3：腾讯云COS图片上传 ✅

**目标**：实现图片上传至COS，返回公开访问URL。

**参考**：04-详细方案设计.md 4.1.3

**任务**：
1. `service/CosClientService.java`：调用腾讯云COS SDK上传图片
2. 传入图片字节 + 文件名，返回公开URL
3. 验证：上传一张测试图片，浏览器访问URL确认可打开

**交付**：COS上传模块可独立调用，返回可访问的URL。

> ⏸️ **检查点 3**：COS上传完成后暂停，用户验证URL可访问。

---

### 步骤 4：智谱OCR引擎A ✅

**目标**：实现智谱OCR调用，输入图片URL返回结构化字段。

**参考**：04-详细方案设计.md 4.1.4

**任务**：
1. `service/OcrZhipuService.java`：调用 `https://open.bigmodel.cn/api/paas/v4/layout_parsing`
2. 传入COS图片URL，返回运单号、收件人、寄件人等字段
3. 记录识别耗时到 ocr_log
4. 验证：用COS URL测试，确认能返回结构化 JSON

**交付**：`OcrZhipuService.java` 可独立调用，返回正确识别结果。

> ⏸️ **检查点 4**：智谱OCR完成后暂停，用户用真实面单照片验证识别效果。

---

### 步骤 5：阿里云OCR引擎B

**目标**：实现阿里云面单OCR调用，返回字段 + 逐字段置信度。

**参考**：04-详细方案设计.md 4.1.5

**任务**：
1. `service/OcrAliyunService.java`：调用阿里云 RecognizeWaybill API
2. 传入COS图片URL，返回运单号、收件人、寄件人等字段
3. 返回 `confidence` 字典，含各字段的 `valueProb` 置信度
4. 记录识别耗时到 ocr_log
5. 验证：用同一张COS URL测试，对比智谱结果

**交付**：`OcrAliyunService.java` 可独立调用，返回结果含置信度。

> ⏸️ **检查点 5**：阿里云OCR完成后暂停，用户对比双引擎识别结果。

---

### 步骤 6：交叉验证引擎（核心业务逻辑） ✅

**目标**：实现双引擎并行调用 + 逐字段交叉比对。

**参考**：04-详细方案设计.md 4.1.6

**任务**：
1. `service/OcrCrossValidatorService.java`：
   - `CompletableFuture` 并行调用引擎A和引擎B
   - 逐字段比对：一致 -> matched，不一致 -> conflict
   - 冲突字段用阿里云 valueProb 仲裁（≥80 优先阿里云）
   - 单引擎失败 -> 优先引擎A（智谱准确率更高）
   - 双引擎失败 -> 转人工
2. 返回结构：`{action, data, diff_fields, confidence}`
3. 验证：用多张照片测试，覆盖一致/差异/失败三种场景

**交付**：交叉验证引擎可独立调用，正确处理三种场景。

> ⏸️ **检查点 6**：交叉验证引擎完成后**必须暂停**，这是核心业务逻辑，用户需检查比对策略和仲裁规则。

---

### 步骤 7：DeepSeek LLM分析 ✅

**目标**：实现语义校验 + 退货原因提取 + 智能分类。

**参考**：04-详细方案设计.md 4.1.7

**任务**：
1. `service/LlmAnalyzerService.java`：调用 DeepSeek V4 Flash API
2. 传入OCR结果，一次完成三项分析：
   - 语义校验（运单号前缀与快递公司匹配等）
   - 退货原因提取（从手写/备注中提取）
   - 智能分类（分到7个标准类别）
3. 返回结构：`{validation, return_reason, category, confidence}`
4. 验证：传入含手写备注的OCR结果，验证LLM分析输出

**交付**：LLM分析模块可独立调用，返回正确的分析结果。

> ⏸️ **检查点 7**：LLM分析完成后暂停，用户检查分析结果准确性。

---

### 步骤 8：数据校验

**目标**：实现运单号格式校验 + 交叉验证状态检查。

**参考**：04-详细方案设计.md 4.2.1

**任务**：
1. `service/ValidatorService.java`：
   - 运单号必填 + 格式校验（8-30位字母数字）
   - 交叉验证状态检查（manual/review/accept）
   - 返回 `(bool, str)` 校验结果
2. 验证：传入各种边界数据测试校验逻辑

**交付**：校验模块可独立调用。

> ⏸️ **检查点 8**：校验逻辑完成后暂停，用户检查校验规则。

---

### 步骤 9：飞书写入 ✅

**目标**：实现确认后的记录写入飞书多维表格（含photo_url、退货原因、分类）。

**参考**：04-详细方案设计.md 4.3

**任务**：
1. `feishu/FeishuAuthService.java`：token 获取 + 缓存 + 自动刷新
2. `feishu/FeishuClientService.java`：batch_create 写入飞书多维表格（含 photo_url、return_reason、return_category 字段）
3. `feishu/FeishuBotService.java`：写入成功后发送飞书机器人卡片通知（运单号+退货原因+分类+查看照片按钮）
4. 写入成功后回填 `feishu_record_id`，更新 MySQL status=synced
5. 验证：写入一条测试记录（含COS图片URL和LLM分析结果），在飞书表格中确认，群内收到通知

**交付**：飞书写入功能可用。

> ⏸️ **检查点 9**：飞书写入完成后暂停，用户在飞书表格中确认数据和图片链接正确。

---

### 步骤 10：API接口

**目标**：实现上传、确认、查询三个 API 接口。

**参考**：06-API接口设计.md

**任务**：
1. `controller/UploadController.java`：`POST /api/upload`，接收图片 -> 上传COS -> 调交叉验证 -> 调LLM分析 -> 存MySQL -> 返回结果
2. `controller/ConfirmController.java`：`POST /api/confirm`，接收确认数据 -> 写入飞书 -> 更新MySQL
3. `controller/RecordsController.java`：`GET /api/records`，从MySQL查询退货记录列表
4. 响应格式包含交叉验证状态 + LLM分析结果
5. 验证：用 curl/Postman 测试三个接口

**交付**：三个 API 接口可用，响应格式符合 06 文档。

> ⏸️ **检查点 10**：API接口完成后暂停，用户检查接口响应格式。

---

### 步骤 11：前端页面 ✅

**目标**：实现拍照上传 + 识别结果展示 + 差异高亮 + LLM分析展示 + 确认写入。

**参考**：04-详细方案设计.md 4.4

**任务**：
1. `Upload.vue`：图片上传组件（支持拖拽、多选）
2. 识别结果展示：表格形式，可编辑
3. **差异高亮**：交叉验证有差异的字段标红，显示两个引擎的不同值
4. 交叉验证状态展示：✅一致 / ⚠️需复核 / ❌转人工
5. **DeepSeek AI分析展示**：退货原因 + 分类 + 置信度 + 语义校验结果
6. 确认按钮：点击后调用 `/api/confirm`
7. `Records.vue`：历史记录列表
8. 底部统计：今日已录入 / 交叉验证通过 / 需复核

**交付**：前端页面完整，端到端流程可走通。

> ⏸️ **检查点 11**：前端完成后暂停，用户检查差异高亮和AI分析展示。

---

### 步骤 12：端到端联调与测试 ✅

**目标**：完整流程联调，抽样测试准确率。

**参考**：08-实施计划与风险.md

**任务**：
1. 端到端联调：拍照 -> COS -> 双引擎OCR -> 交叉验证 -> LLM分析 -> 确认 -> 飞书
2. 抽样 100 张真实面单测试，统计：
   - 交叉验证一致率（目标 ~88%）
   - 差异需复核率（目标 ~10%）
   - 转人工率（目标 ~2%）
   - LLM退货原因提取覆盖率
3. 修复联调中发现的问题
4. 部署上线

**交付**：系统上线运行，准确率达标。

> ⏸️ **检查点 12**：联调完成后暂停，用户确认准确率达标后上线。
