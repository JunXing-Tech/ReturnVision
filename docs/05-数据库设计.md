# 数据库设计

#### 4.5.1 为什么需要本地数据库

退货OCR的最终数据写入飞书多维表格，但本地数据库在**识别到确认**这段流程中不可或缺：

| 理由 | 说明 |
|------|------|
| 待确认记录暂存 | OCR 识别后、客服确认前，需持久化识别结果；客服可能关闭浏览器后回来继续 |
| 状态流转管理 | 记录从"待确认 -> 已确认 -> 已同步飞书"的全生命周期状态 |
| 识别质量监控 | 记录 OCR 引擎、置信度、耗时，用于监控准确率、优化引擎选择 |
| 历史查询加速 | 本地缓存退货记录，查询历史不必每次调飞书 API |

#### 4.5.2 表结构设计

本项目设计 **2 张表**，精简够用：

**表1：return_records（退货记录表）** - 核心表

```sql
CREATE TABLE return_records (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    waybill_no      VARCHAR(30),                    -- 运单号
    rec_name        VARCHAR(50),                    -- 收件人姓名
    rec_phone       VARCHAR(20),                    -- 收件人电话
    rec_address     TEXT,                           -- 收件人地址
    sender_name     VARCHAR(50),                    -- 寄件人姓名
    sender_phone    VARCHAR(20),                    -- 寄件人电话
    express_company VARCHAR(30),                    -- 快递公司
    return_date     DATE,                           -- 退货日期
    status          VARCHAR(20) DEFAULT 'pending',  -- 状态：pending/confirmed/synced/failed
    ocr_engine      VARCHAR(20),                    -- 识别引擎：zhipu/aliyun/cross_validated/manual
    ocr_confidence  DECIMAL(3,2),                   -- OCR置信度（0.00-1.00）
    image_url       VARCHAR(500),                   -- 腾讯云COS图片URL
    return_reason   VARCHAR(100),                   -- 退货原因（DeepSeek分析）
    return_category VARCHAR(50),                    -- 退货分类：质量问题/物流问题/尺寸不符/...
    llm_confidence  DECIMAL(3,2),                   -- LLM分析置信度（0.00-1.00）
    feishu_record_id VARCHAR(50),                   -- 飞书记录ID（写入后回填）
    remark          TEXT,                           -- 备注
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,  -- 识别时间
    confirmed_at    DATETIME,                       -- 确认时间
    synced_at       DATETIME                        -- 同步飞书时间
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE INDEX idx_status ON return_records(status);       -- 按状态筛选（待确认列表）
CREATE INDEX idx_date ON return_records(return_date);    -- 按日期查询
```

**表2：ocr_log（OCR识别日志表）** - 识别监控

```sql
CREATE TABLE ocr_log (
    id            INT PRIMARY KEY AUTO_INCREMENT,
    record_id     INT,                            -- 关联退货记录ID
    engine        VARCHAR(20) NOT NULL,           -- 引擎：zhipu_ocr/aliyun_waybill
    duration_ms   INT,                            -- 识别耗时（毫秒）
    success       TINYINT(1) DEFAULT 1,           -- 是否成功
    confidence    DECIMAL(3,2),                   -- 置信度
    error_msg     TEXT,                           -- 错误信息
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 4.5.3 状态流转设计

return_records 的 status 字段管理记录全生命周期：

```
[上传图片] -> [OCR识别]
                  │
        ┌─────────┴─────────┐
        ▼                   ▼
   识别成功              识别失败
   status=pending       status=failed
        │                   │
        ▼                   ▼
   客服确认/修改        手动录入
   status=confirmed     status=confirmed
        │                   │
        ▼                   ▼
   写入飞书表格
   status=synced
   回填 feishu_record_id
```

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| pending | 待确认 | OCR识别成功，等待客服确认 |
| confirmed | 已确认 | 客服确认或手动录入 |
| synced | 已同步 | 已写入飞书多维表格 |
| failed | 识别失败 | OCR引擎均无法识别 |

#### 4.5.4 ER 关系图

```
┌─────────────────────┐         ┌─────────────────────┐
│  return_records      │         │  ocr_log             │
│─────────────────────│         │─────────────────────│
│  id (PK)            │◀────────│  record_id (FK逻辑)  │
│  waybill_no         │  一对多  │  engine              │
│  rec_name           │         │  duration_ms         │
│  status             │         │  success             │
│  ocr_engine         │         │  confidence          │
│  ocr_confidence     │         │  error_msg           │
│  image_url          │         └─────────────────────┘
│  return_reason      │
│  return_category    │
│  llm_confidence     │
│  feishu_record_id   │
│  created_at         │
│  confirmed_at       │
│  synced_at          │
└─────────────────────┘
```

> 一条退货记录会同时经历两个引擎OCR（智谱+阿里云并行），因此 ocr_log 与 return_records 为一对多关系。

#### 4.5.7 数据流转总结

| 操作 | 源 | 目标 | 触发时机 |
|------|----|------|---------|
| 识别结果暂存 | OCR引擎返回 | return_records（status=pending） | 上传图片识别完成，含COS图片URL |
| 识别日志记录 | OCR引擎执行 | ocr_log | 每次OCR调用（双引擎各记一条） |
| 确认写入 | 客服确认 | return_records（status=confirmed） | 客服点击确认 |
| 同步飞书 | 确认后记录 | 飞书多维表格 + return_records（status=synced） | 确认后自动/手动触发 |
| 历史查询 | return_records | 前端列表 | 客服查看历史记录 |
