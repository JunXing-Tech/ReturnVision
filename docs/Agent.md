## 任务 001：退货OCR解决方案设计

### 任务时间
2026-07-08

### 任务要求
1. 根据 `退货.md` 需求文档，设计可执行落地的解决方案
2. 核心需求：拍快递单照片 -> OCR提取信息 -> 写入飞书表格
3. 联网搜索验证技术方案真实性
4. 参考前两个方案的文档结构与代码规范

### 关键决策记录

#### 决策1：OCR主引擎 - 阿里云电子面单识别
- **选择**：阿里云 RecognizeWaybill
- **理由**：专用快递面单优化，准确率93%，支持图像增强（旋转/畸变/模糊），200次/月免费
- **验证**：文档2025-12-09更新，活跃维护

#### 决策2：OCR备引擎 - GLM-5.2多模态
- **选择**：GLM-5.2视觉能力
- **理由**：主引擎失败时补充；可直接输出结构化JSON；灵活提取任意字段
- **备选**：GPT-4o（清晰文档近完美但含噪下降明显）、GLM-4V-Flash（免费）

#### 决策3：双引擎策略
- 主引擎识别成功 -> 直接返回结果
- 主引擎失败/置信度低 -> 自动切换GLM-5.2多模态
- 两者都失败 -> 转人工录入

#### 决策4：人工确认环节
- OCR结果返回前端，客服确认/修改后再写入飞书
- 保证数据质量，OCR非100%准确需人工兜底

#### 决策5：飞书写入 - 复用已验证的飞书多维表格API
- batch_create接口，单次最多500条
- lark-oapi Python SDK

### 联网验证结果

## 任务 002：数据库设计

### 任务时间
2026-07-08

### 任务要求
1. 明确退货OCR业务是否需要数据库
2. 设计数据库表结构

### 决策
1. **需要数据库**：飞书写入是最终存储，但识别到确认的中间流程需要本地库；4个理由：待确认记录暂存、状态流转管理、识别质量监控、历史查询加速
2. **2张表设计**：
   - return_records（退货记录表）：核心表，含运单号/收寄件人信息/状态(status)/OCR引擎/置信度/飞书记录ID
   - ocr_log（OCR识别日志表）：记录每次OCR调用的引擎/耗时/成功/置信度/错误
3. **状态流转**：pending(待确认) -> confirmed(已确认) -> synced(已同步飞书)；识别失败为 failed
4. **一对多关系**：一条退货记录可能经历两次OCR（主引擎失败切备引擎），ocr_log与return_records为一对多

### 产出
- `退货OCR解决方案.md` 更新至 v1.1，新增 4.5 数据库设计（含DDL/状态流转图/ER图/ORM/数据流转）
- 项目目录结构更新：models/ 增加 database.py

## 任务 005：双引擎交叉验证架构升级

### 任务时间
2026-07-10

### 任务要求
1. 阿里云OCR准确率仅93%，需保底方案确保识别准确性
2. 验证智谱OCR解析工具能否作为引擎A
3. 验证GLM-5.2多模态能否完成兜底
4. 联网搜索验证方案正确性和可执行性

### 关键决策记录

#### 决策1：修正GLM-5.2错误
- **问题**：GLM-5.2为纯文本模型，不支持图片输入，原方案无法执行
- **修正**：改用智谱OCR解析工具（GLM-OCR）作为引擎A

#### 决策2：智谱OCR为引擎A
- **选择**：智谱OCR解析工具（layout_parsing API）
- **理由**：OmniDocBench v1.5第一（94.62%），0.01元/次（阿里云1/8），支持手写体和JSON Schema
- **验证**：docs.bigmodel.cn 官方文档活跃维护

#### 决策3：阿里云OCR为引擎B
- **选择**：阿里云RecognizeWaybill
- **理由**：93%准确率，200次/月免费，返回逐字段valueProb置信度
- **关键发现**：阿里云API返回valueProb置信度，原方案未利用

#### 决策4：双引擎交叉验证策略
- **策略**：两引擎并行识别 + 逐字段比对
- **一致**（~88%）：自动采用，准确率接近99.9%
- **部分差异**（~10%）：高亮差异字段，客服只核对差异
- **完全不一致**（~2%）：转人工录入
- **原理**：两独立引擎犯同样错误的概率极低

#### 决策5：四层保障体系
1. 双引擎交叉验证（自动）
2. 阿里云置信度辅助（自动，valueProb ≥ 80优先）
3. 运单号格式校验（自动）
4. 人工确认（必经）

### 联网验证结果
| 验证项 | 状态 | 结论 |
|--------|------|------|
| 阿里云OCR返回置信度 | ✅ 确认 | 返回valueProb（0-100），原方案未利用 |
| GLM-5.2支持图片 | ❌ 不支持 | 纯文本模型，已修正为智谱OCR |
| 智谱OCR准确率 | ✅ 94.62% | OmniDocBench v1.5第一 |
| 智谱OCR价格 | ✅ 0.01元/次 | 阿里云的1/8 |
| GLM-4V-Flash免费 | ✅ 免费 | 可作为备选多模态引擎 |
| 智谱OCR支持JSON Schema | ✅ 支持 | 可直接提取结构化字段 |
