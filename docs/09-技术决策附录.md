# 技术决策附录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| OCR引擎A | 智谱OCR解析工具（GLM-OCR） | OmniDocBench v1.5第一（94.62%），0.01元/次，支持手写体和JSON Schema；单引擎降级时优先 |
| OCR引擎B | 阿里云电子面单识别 | 93%准确率，200次/月免费，返回逐字段valueProb置信度，专用面单优化 |
| 交叉验证策略 | 双引擎并行+字段比对 | 两独立引擎一致时准确率接近99.9%，覆盖~88%场景 |
| 置信度利用 | 阿里云valueProb ≥ 80优先 | 差异字段用阿里云置信度辅助判断哪个引擎更可信 |
| 数据库 | MySQL 8.0 | 生产级可靠性，公司已有基础设施；存储待确认记录、OCR日志、状态流转 |
| 图片存储 | 腾讯云COS | 持久化存储，URL供双引擎OCR和飞书引用；50GB/月免费额度 |
| LLM分析 | DeepSeek V4 Flash | 语义校验+退货原因+智能分类一次完成；~0.0004元/次；1M上下文 |
| 写入目标 | 飞书多维表格 | 结构化存储，API完善，支持协同查看，含photo_url、退货原因、分类字段 |
| 人工确认 | 必须确认后写入 | 五层保障最后一层，确保数据质量 |
| 差异字段展示 | 前端高亮差异 | 客服只核对差异字段，平均~3秒/单，效率提升10-20倍 |
| 批量上传 | 支持 | 退货量大时提升效率（注意智谱并发限制为2） |
| 架构模式 | 前后端分离 + MVC | 职责清晰，便于维护 |
| 后端语言 | Java 21 + Spring Boot 3.x | 企业级稳定性，团队技术栈统一，生态成熟 |
| ORM框架 | MyBatis-Plus | 国内主流，灵活SQL控制，代码生成便捷 |
